## Migration Pipeline

### Automated Steps (migration.py)

The migration.py script orchestrates the complete conversion:

```bash
# Activate virtual environment
source .venv/Scripts/activate

# Run migration for 3.0 branch
python migration.py --branch 3.0
```

**Pipeline Steps:**
1. **Step 1: Init** - Initialize docs/ directory structure
2. **Step 2: Scan** - Scan RST files in source/ directory
3. **Step 3: Migrate** - Convert RST to Markdown using petersmythe/translate
4. **Step 4: Nav** - Generate navigation structure (nav_generated.yml)
5. **Step 5: Postprocess** - Standard postprocessing (mkdocs_translate)
6. **Step 6: Interpreted Text** - Convert Sphinx roles to Markdown links
7. **Step 7: Frontmatter** - Add render_macros: true for variable files

## Known Conversion Issues and Fixes

### 1. Image Path Issues (CRITICAL)

**Problem:** Conversion tool creates incorrect image paths (absolute, over-corrected, wildcards)

**Fix Scripts (run in order):**
1. `fix_image_paths.py` - Convert absolute paths to relative
2. `fix_anchor_case.py` - Fix anchor case sensitivity
3. `fix_variable_substitution.py` - Replace ##SUBST## placeholders
4. `fix_overcorrected_paths.py` - Remove excessive ../
5. `fix_all_image_paths.py` - Comprehensive multi-strategy fix
6. `fix_wildcard_images.py` - Replace .* with .svg
7. `fix_ysld_image_paths.py` - Fix YSLD reference paths

**Expected Result:** 100% of image issues resolved (2,071 fixed in 2.28.x)

### 2. Broken Anchor Links (CRITICAL)

**Problem:** Cross-document references, workshop links, incorrect anchor names

**Fix Scripts (run in order):**
1. `fix_broken_anchors.py` - Fix RST-style cross-document references
2. `fix_all_broken_anchors.py` - Fix cross-document and workshop anchors
3. `fix_remaining_anchors.py` - Fix remaining broken anchors
4. `fix_final_anchors.py` - Fix final broken anchors with specific mappings

**Expected Result:** 100% of anchor issues resolved (128 fixed in 2.28.x)

### 3. Grid Card Titles (HIGH PRIORITY)

**Problem:** Conversion tool creates malformed titles like "Programming GuideConfigIndex"

**Fix Script:**
- `fix_grid_card_titles.py` - Extract clean titles from RST source

**Expected Result:** 669 titles fixed in 121 files (2.28.x)

**Example Fixes:**
- `[Programming GuideConfigIndex](config/index.md)` → `[Config](config/index.md)`
- `[ExtensionsAuthkeyIndex](authkey/index.md)` → `[Authkey](authkey/index.md)`

### 4. Macro Rendering (HIGH PRIORITY)

**Problem:** Files with {{ version }} or {{ release }} need render_macros: true frontmatter

**Fix Script:**
- `fix_macro_rendering.py` - Add frontmatter to files with macros

**Note:** migration.py Step 7 handles this automatically, but script available for post-conversion fixes

### 5. Include Statement Issues (MEDIUM PRIORITY)

**Problem:** Multi-line includes, invalid parameters, includes in code blocks

**Fix Scripts:**
1. `fix_include_syntax.py` - Convert multi-line to single-line
2. `fix_include_paths.py` - Fix paths relative to docs directory
3. `fix_all_include_issues.py` - Wrap includes in code blocks with {%raw%}
4. `fix_include_with_params.py` - Fix invalid Jinja2 start/end parameters

**Common Issues:**
- Multi-line `{% include %}` causes macro syntax errors
- Includes in code blocks get processed (need {%raw%} tags)
- Paths outside docs directory fail (need to comment out)

### 6. Tilde Code Fences (MEDIUM PRIORITY)

**Problem:** MkDocs requires backticks (```), not tildes (~~~)

**Fix Script:**
- `fix_tilde_fences.py` - Replace all ~~~ with ```

**Impact:** Prevents visible ~~~ characters in rendered pages

### 7. Blank Header Tables (MEDIUM PRIORITY)

**Problem:** Tables with blank first row render incorrectly

**Fix Script:**
- `fix_blank_header_tables.py` - Remove blank first rows

**Expected Result:** 212 tables fixed in 104 files (2.28.x)

**Common Locations:** SLD reference, cookbook, workshop, filter docs

### 8. Missing Version/Release Macros (LOW PRIORITY)

**Problem:** Conversion tool drops |version| and |release| in ~79 locations

**Fix Script:**
- `fix_missing_version_macros.py` - Restore macros in installation instructions

**Check Script:**
- `check_missing_version_macros.py` - Identify missing macros

**Common Patterns:**
- "example: 2.28.0" → "example: {{ release }}"
- "geoserver-2.28-" → "geoserver-{{ version }}-"

### 9. User Manual Index Format (CRITICAL)

**Problem:** Conversion tool uses definition list format instead of grid cards

**Manual Fix Required:**
- Update `doc/en/user/docs/index.md` to use grid cards format
- Reference: `doc/en/developer/docs/index.md` (correct format)

### 10. Responsive Navigation (MEDIUM PRIORITY)

**Problem:** Horizontal navigation tabs overflow on mobile screens

**Solution:** Already configured in mkdocs.yml from 2.28.x reference

## Common Pitfalls

### 1. Running Scripts Before Conversion
**Problem:** Fix scripts expect converted Markdown files  
**Solution:** Run migration.py FIRST, then run fix scripts

### 2. Forgetting to Activate venv
**Problem:** Python dependencies not found  
**Solution:** Always run `source .venv/Scripts/activate` first

### 3. Copying 2.28.x-Specific Content
**Problem:** Chinese docs, PDF config don't exist in 3.0  
**Solution:** Use 2.28.x as reference, but adapt for 3.0 differences

### 4. Not Merging nav_generated.yml
**Problem:** Navigation structure not updated in mkdocs.yml  
**Solution:** Merge nav_generated.yml into mkdocs.yml, then delete nav_generated.yml

### 5. Skipping Image/Anchor Fixes
**Problem:** 2,000+ broken images, 100+ broken anchors  
**Solution:** Run ALL fix scripts in Phase 1 (critical fixes)
