## Key Insights

### Include Directive Behavior
The 2.28.x branch uses the standard MkDocs `include` directive which:
- Processes files from the docs root directory
- Links in included files are resolved relative to where they're included, not where the include file is located
- This is different from 3.0's `include-markdown` directive which uses relative paths from the include file

### API Template Variables
The remaining "warnings" about `{{ api_url }}` template variables are expected:
- These are Jinja2 template variables that resolve at build time
- They point to external API specification files
- MkDocs reports them as "not found among documentation files" but they work correctly
- The 3.0 branch removed the `api_url` configuration to eliminate these informational messages
