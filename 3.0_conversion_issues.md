## PR Description Notes

**IMPORTANT: Include in PR description:**

This conversion required fixing 266 interpreted text role occurrences that were not converted by the initial migration tool. The roles used Pandoc's attribute syntax (`` `text`{.interpreted-text role="role_name"} ``) rather than standard Markdown links.

Key roles fixed:
- `nightly_community` / `nightly_extension` → Build server download links
- `doc` → Cross-references to other documentation pages
- `ref` → Internal anchor links
- `wiki`, `docguide`, `developer`, `geotools`, `github` → External documentation links
- `download_release`, `download_pending` → SourceForge download links
- `abbr` → Abbreviations (converted to bold text)

This was a 3.0-specific issue not present in the 2.28.x branch conversion, likely due to differences in how the conversion tool handled these roles between branches.
