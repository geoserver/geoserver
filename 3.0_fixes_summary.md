# 3.0 Branch Post-Conversion Fixes Summary

This document tracks all fixes applied to the 3.0 branch after the RST-to-Markdown conversion.

## Completed Fixes

### 1. Broken Download Links (51 files)
**Script:** `fix_download_links.py`  
**Date:** 2026-03-04  
**Commit:** c08b67f176

**Issue:** The conversion tool incorrectly converted RST interpreted text roles (`:download_extension:`, `:nightly_extension:`) to broken URLs pointing to directory paths instead of actual ZIP files.

**Fix:**
- Added `snapshot` macro variable to `doc/version.py` (e.g., `3.0-SNAPSHOT`)
- Release links now point to SourceForge: `https://sourceforge.net/projects/geoserver/files/GeoServer/{{ release }}/extensions/geoserver-{{ release }}-plugin-name-plugin.zip`
- Snapshot links now point to build server: `https://build.geoserver.org/geoserver/main/extensions/geoserver-{{ snapshot }}-plugin-name-plugin.zip`

**Files affected:**
- All styling extension installations (CSS, YSLD, MBStyle)
- All OGC service installations (WPS, WCS, CSW, OGC API Features)
- All database connector installations (MySQL, Oracle, SQL Server)
- All raster format extensions (GDAL, ArcGrid, Image Pyramid)
- 40+ other extension installations

**GitHub Actions warnings fixed:**
- All broken download links in extension/community installation pages
